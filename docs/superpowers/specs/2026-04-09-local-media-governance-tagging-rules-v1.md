# 本地离线媒体治理系统 标签生成规则 v1

- 日期: 2026-04-09
- 状态: Draft

## 1. 目标

本文件定义第一版标签生成规则，解决以下问题：

- AI 应该输出什么层次的标签
- `raw_tags / canonical_tags / structured_attributes` 如何协同
- 敏感内容如何直白表达但不失控
- 标签如何兼顾表达力、稳定性与可维护性

## 2. 总体原则

第一版标签策略遵循：

- `内容理解优先`
- `表达直白明确`
- `保留原始语义`
- `宽松归一化`
- `不改变原意`

标签系统不是为了“说得好听”，而是为了：

- 让系统理解内容
- 让用户筛选资源
- 让候选审核更高效

## 3. 三层输出模型

第一版统一采用三层输出：

1. `raw_tags`
2. `canonical_tags`
3. `structured_attributes`

## 3.1 `raw_tags`

含义：

- 模型对内容最直接的原始表达

要求：

- 保留语义细节
- 允许直白敏感表达
- 不强制收缩到固定词表
- 尽量短词或短短语，不要写成长句

示例：

- `自拍做爱`
- `单人写真`
- `室内床上`
- `手机竖屏自拍`
- `聊天记录截图`
- `低清转码`

## 3.2 `canonical_tags`

含义：

- 面向筛选与统计的规范标签候选

要求：

- 从 `raw_tags` 中宽松归一化
- 只合并明显同义表达
- 不能改变原意

示例：

- `手机自拍视频` -> `手机自拍`
- `竖版自拍` -> `竖屏自拍`
- `成人视频` -> `AV` 或反之  
  取决于最终 canonical 约定，但必须一致

## 3.3 `structured_attributes`

含义：

- 适合稳定筛选和排序的结构化属性

第一版建议输出：

- `media_type`
- `subject_count`
- `capture_type`
- `orientation`
- `has_face`
- `is_sensitive`
- `quality_tier`
- `resolution_tier`

这些字段不是自由标签，而是结构化判断结果。

## 4. 标签粒度规则

第一版标签粒度要求：

- 不要只给过于泛化的大类
- 也不要碎到接近自然语言描述

合适的粒度应接近：

- `单人写真`
- `泳装写真`
- `自拍做爱`
- `室内`
- `竖屏自拍`
- `聊天记录截图`

不理想的例子：

- 过粗：`图片`、`视频`、`人物`
- 过细：`一个穿白色衣服的人在室内床边自拍`

后者应属于摘要，而不是标签。

## 5. 敏感内容策略

第一版敏感内容标签要求：

- `直白明确`
- `不做含糊 euphemism`

原因：

- 用户需要靠这些标签筛选
- 模糊表达会降低治理价值

### 5.1 允许的表达方向

例如：

- `AV`
- `做爱`
- `口交`
- `自慰`
- `裸体`
- `情侣性爱`
- `自拍做爱`

### 5.2 不建议的表达方向

例如：

- `成人内容`
- `暧昧内容`
- `亲密行为`

这些表达过于抽象，不利于筛选和审核。

### 5.3 敏感标签与其他标签并存

敏感标签不应替代普通内容标签。

例如：

- `自拍做爱`
- `室内`
- `竖屏自拍`
- `低清`

这几类标签可以同时存在。

## 6. 场景标签与拍摄方式标签

第一版不必把所有维度都拆成独立 namespace。

建议：

- 先让 `content` 承担大多数内容和场景标签
- 把部分稳定属性下沉到 `structured_attributes`

例如：

- `自拍`  
  可同时作为 raw/canonical tag，也可映射到 `capture_type=selfie`

- `竖屏`  
  可同时作为 raw/canonical tag，也可映射到 `orientation=portrait`

## 7. 图片与视频的标签差异

## 7.1 图片

图片标签更偏静态内容：

- 主体
- 场景
- 拍摄方式
- 风格
- 质量

## 7.2 视频

视频标签更偏汇总结果：

- 主体
- 行为
- 场景
- 拍摄方式
- 内容类型
- 质量

视频标签来自多帧汇总，不来自单帧偶然判断。

## 8. 标签生成流程

建议流程：

1. Worker 生成 `raw_tags`
2. Worker 生成 `structured_attributes`
3. 主服务或规范化层生成 `canonical_tags`
4. `tags + file_tags` 写入正式标签关系
5. `analysis_results` 保存原始分析快照

## 9. 规范化规则

第一版规范化规则只做这几类操作：

- 同义词合并
- 词序轻微归一
- 明显冗余修饰词裁剪
- 中英文等义映射

不做：

- 改变语义主体
- 把上下位概念硬合并
- 把不同动作或不同场景视为同义

## 10. 标签来源优先级

同一个文件、同一个标签可有多来源：

- `human`
- `rule`
- `ai`

前端展示优先级：

1. `human`
2. `rule`
3. `ai`

这样可以保留证据，同时以人工修正为准。

## 11. 最小 Prompt 原则

第一版标签生成 prompt 需要明确要求模型：

- 输出短标签而不是长句
- 可以直白表达敏感内容
- 不要刻意回避用户真实筛选需求
- 同时输出结构化属性
- 避免空泛泛标签

建议 prompt 明确要求：

- “标签优先使用短词或短短语”
- “敏感内容允许直白标签”
- “不要用模糊 euphemism”
- “摘要和标签分开输出”

## 12. 典型例子

## 12.1 图片示例

输入：

- 单人室内竖版写真，半身构图，清晰

建议输出：

- `raw_tags`: `["单人写真", "室内", "竖屏", "半身"]`
- `canonical_tags`: `["单人写真", "室内", "竖屏"]`
- `structured_attributes`:
  - `subject_count=1`
  - `orientation=portrait`
  - `capture_type=photo`
  - `is_sensitive=false`

## 12.2 视频示例

输入：

- 手机竖屏自拍视频，室内，双人做爱，清晰度中等

建议输出：

- `raw_tags`: `["自拍做爱", "室内", "竖屏自拍", "双人"]`
- `canonical_tags`: `["自拍做爱", "室内", "竖屏自拍"]`
- `structured_attributes`:
  - `subject_count=2`
  - `capture_type=selfie`
  - `orientation=portrait`
  - `is_sensitive=true`

## 13. 第一版边界

第一版不追求：

- 完整标签本体论
- 复杂层级树形 taxonomy
- 完美的跨语种统一

第一版要做到：

- 标签直白可用
- 规范化不过度
- 结构化属性稳定
- 对敏感内容表达不回避

## 14. 当前结论

第一版标签系统的关键不是“词表多大”，而是：

- AI 能否真实理解内容
- 是否保留原始语义
- 是否能稳定筛选
- 是否方便人工修正

因此标签系统应始终坚持：

- 原始标签保留
- 规范标签宽松归一
- 结构化属性稳定落库
