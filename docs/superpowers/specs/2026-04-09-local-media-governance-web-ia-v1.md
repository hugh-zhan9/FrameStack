# 本地离线媒体治理系统 Web IA v1

- 日期: 2026-04-09
- 状态: Draft

## 1. 目标

本文件定义第一版 Web 工作台的信息架构和核心页面职责。

目标不是设计视觉细节，而是明确：

- 用户进入系统后先看到什么
- 一级导航如何分工
- 候选审核与资源浏览如何分离
- 单文件详情如何承载信息和操作

## 2. 首页定位

第一版首页不是普通图库首页，而是：

- `治理驾驶舱`

首页优先回答：

- 当前总共有多少资源
- 哪些候选最值得处理
- 当前任务进展如何
- 最近有哪些变化

## 3. 首页模块

首页建议包含：

### 3.1 资源总览

- 图片总数
- 视频总数
- 总占用空间
- 在线 volume 数

### 3.2 待处理候选

- 完全重复候选数
- 同内容候选数
- 同一人候选数
- 同一系列候选数
- `trash_candidate` 数量

### 3.3 任务状态

- 扫描中任务数
- 分析中任务数
- 失败任务数
- 最近完成任务数

### 3.4 近期变化

- 最近新增文件数
- 最近移动/重命名文件数
- 最近进入待删候选的文件数

### 3.5 快捷入口

- 进入重复候选审核
- 进入同一人候选审核
- 进入同一系列候选审核
- 进入待删审核
- 进入全部资源浏览

## 4. 一级导航

第一版建议一级导航：

1. `首页`
2. `候选审核`
3. `资源浏览`
4. `任务中心`
5. `设置`

职责分工：

- `首页`: 总览和分发
- `候选审核`: 治理主流程
- `资源浏览`: 全量搜索和筛选
- `任务中心`: 可观测性与恢复
- `设置`: 目录、模型、调度和系统选项

## 5. 候选审核

## 5.1 子视图

第一版 `候选审核` 下分四个子视图：

1. `同内容`
2. `同一人`
3. `同一系列`
4. `待删候选`

原因：

- 四类候选的语义不同
- 审核动作不同
- 混在一起会显著增加认知负担

## 5.2 布局

第一版采用：

- `左侧候选组列表`
- `右侧当前组详情`

不采用“大平铺审核”方式。

## 5.3 候选组详情内容

建议展示：

- 组类型
- 组置信度
- 成员数量
- 代表封面
- 成员缩略图或视频封面
- 名称与路径
- 标签
- 描述摘要
- 质量评分
- 推荐理由
- 推荐保留项 / best_quality

## 5.4 候选审核动作

建议支持：

- 保留某个文件
- 标记某个文件为 `trash_candidate`
- 忽略某个成员
- 忽略整个候选组
- 确认候选关系成立

后续增强可补：

- 手动拆组
- 手动合组
- 手动调整组内排序

## 6. 资源浏览

## 6.1 页面定位

`资源浏览` 用于：

- 全量资源浏览
- 搜索
- 多维筛选
- 单文件查看和快速操作

## 6.2 媒体视角

默认策略：

- `默认混合浏览`
- 提供 `全部 / 图片 / 视频` 切换

## 6.3 展示方式

第一版采用：

- 照片管理器式网格布局

## 6.4 筛选风格

第一版筛选风格：

- `专业型`

不是大众化简化图库。

建议支持：

- 名称
- 路径
- 标签
- 媒体类型
- 文件状态
- 质量等级
- 是否敏感
- 是否有人脸
- 候选状态
- 分辨率区间
- 时长区间
- 文件大小区间
- volume
- 最近新增
- 最近变动

其中 `候选状态` 例如：

- 是否属于 `same_person`
- 是否属于 `same_series`
- 是否属于 `same_content`
- 是否属于 `trash_candidate`

## 7. 单文件详情

第一版单文件详情优先采用：

- `右侧详情抽屉`

不要求第一版就建设复杂独立详情页。

详情抽屉建议展示：

- 预览图或视频封面
- 基础元数据
- 标签
- 描述摘要
- 质量评分
- 所属候选组
- 历史路径
- 主要操作按钮

适合：

- 快速浏览
- 快速审核
- 保持资源流上下文

## 8. 任务中心

第一版任务中心用于：

- 查看当前处理进度
- 查看失败任务
- 执行人工重试

默认结构：

- `进行中`
- `等待中`
- `失败 / dead`
- `最近完成`

默认展示：

- 任务列表
- 支持按 volume 过滤

## 9. 设置

第一版设置建议包含：

- 扫描目录 / volume 管理
- 默认 provider 和模型
- 定时扫描策略
- 并发与任务参数
- 敏感标签策略

## 10. 当前结论

第一版 Web IA 的核心不是“页面多漂亮”，而是：

- 信息分层清楚
- 治理主流程顺手
- 候选审核效率高
- 搜索筛选足够专业

因此信息架构上最重要的决定是：

- 首页做治理驾驶舱
- 候选审核独立一级导航
- 资源浏览走专业筛选 + 网格浏览
- 单文件详情优先抽屉化
